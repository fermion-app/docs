The `getSessionId()` method returns the unique **session ID** associated with your active sandbox connection. This ID identifies the live session currently linked to your WebSocket connection and is useful for debugging, tracking, or logging sandbox activity.  

If the sandbox is not connected: for example, before calling `create()` or after calling `disconnect()`, this method will return `null`. The session ID is distinct from the **snippet ID** returned by `create()`; while the snippet ID identifies a persistent sandbox that can be reconnected later, the session ID represents a temporary, live connection that expires once the session ends.  

### `getSessionId()`

**Description:**
Returns the current playground session ID, or null if not connected.

**Parameters:**
None

**Returns:**
`string | null` - Session ID or null

**Example:**
```typescript
console.log(sandbox.getSessionId()) // null

await sandbox.create({ shouldBackupFilesystem: false })
console.log(sandbox.getSessionId()) // 'session-abc123'
```

<Note>
Session ID is different from snippet ID. Session is temporary, snippet is persistent.
</Note>
